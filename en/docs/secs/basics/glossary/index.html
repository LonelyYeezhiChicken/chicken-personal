🔰 SECS/GEM 術語表

本章節彙整 SECS 系列中反覆出現的縮寫與專有名詞。遇到不熟悉的代號，先來這裡查，再點連結深入閱讀。

角色與架構​

術語全名 / 中文說明深入閱讀
Host主機工廠中央控制電腦，發指令、收資料aboutSECS
Equipment設備 / 機台蝕刻機、清洗機等，執行任務並回報aboutSECS
GEMGeneric Equipment ModelSECS-II 之上的設備行為標準（E30）secsAndGem
HSMSHigh-Speed SECS Message Services以 TCP/IP 傳遞 SECS-II 訊息（E37）protocol
SECS-I—以 RS-232 傳遞 SECS-II 訊息（E4）secs1BlockTransfer
SECS-II—訊息內容與格式標準（E5）secsStructure

訊息結構​

術語全名 / 中文說明深入閱讀
SxFyStream × Function訊息代號，如 S1F13secsStructure
Stream (S)訊息主類別S1=狀態、S2=控制、S5=警報…streamOverview
Function (F)功能編號同 Stream 下的具體訊息secsStructure
W-BitWait Bit=1 表示需回覆，=0 表示不需回覆secsStructure
Data Item資料項目List、ASCII、U4、B 等 SECS-II 資料型別secsStructure

狀態變數與常數​

術語全名 / 中文說明深入閱讀
SVIDStatus Variable ID狀態變數編號，可查當前值（S1F3/F4）s1-equipmentStatus
VIDVariable ID變數編號，用於事件報告內容eventReport
ECIDEquipment Constant ID設備常數編號，設定後較少變動（S2F13–F16）s2-equipmentControl
DVVALData Value資料值，常出現在 S6F11 事件內容中s6-dataCollection

事件報告​

術語全名 / 中文說明深入閱讀
CEIDCollection Event ID收集事件編號，如「片盒載入」eventReport
RPTIDReport ID報告編號，定義事件要回報哪些 VIDeventReport
DATAIDData ID事件資料識別碼，出現在 S6F11s6-dataCollection
DRACKDefine Report AcknowledgeS2F34 回覆碼，定義報告是否成功eventReport
LRACKLink Report AcknowledgeS2F36 回覆碼，連結 CEID 是否成功eventReport
ERACKEnable Report AcknowledgeS2F38 回覆碼，啟用事件是否成功eventReport

配方與控制​

術語全名 / 中文說明深入閱讀
PPIDProcess Program ID配方（Process Program）名稱或編號s7-recipe
PPProcess Program設備執行製程用的配方檔s7-recipe
RCMDRemote Command遠端指令名稱，如 START、STOPs2-equipmentControl
HCACKHost Command AcknowledgeS2F42 回覆碼，指令是否被接受s2-equipmentControl
CPNAME / CPVALCommand Parameter Name/ValueS2F41 指令的參數名與值s2-equipmentControl

警報​

術語全名 / 中文說明深入閱讀
ALIDAlarm ID警報編號s5-alarm
ALCDAlarm Code警報代碼，含嚴重度與 Set/Clear 資訊s5-alarm
ALTXAlarm Text警報文字描述s5-alarm
ACKC5Acknowledge Code 5S5F4/S5F8 回覆碼alarmManagement

通訊與狀態​

術語全名 / 中文說明深入閱讀
COMMACKCommunications AcknowledgeS1F14 回覆碼，通訊是否建立communicationState
ONLACKON-LINE AcknowledgeS1F18 回覆碼，上線是否成功controlState
OFLACKOFF-LINE AcknowledgeS1F16 回覆碼，下線是否成功controlState
COMMUNICATING通訊中GEM 通訊狀態，可交換業務訊息communicationState
REMOTE遠端模式ON-LINE 子狀態，Host 可下達控制指令controlState
Processing State處理狀態INIT / IDLE / EXECUTING 等製程狀態processingState

HSMS 控制訊息​

術語全名 / 中文說明深入閱讀
Select.req/rsp選擇請求/回覆HSMS Session 建立握手hsmsConnection
Linktest.req/rsp連線測試HSMS 心跳，確認 TCP 仍存活hsmsConnection
T3Reply Timeout等待 SECS 回覆的逾時（預設 45 秒）hsmsConnection
System Bytes系統位元組HSMS Header 中用於配對請求與回覆hsmsMessage
