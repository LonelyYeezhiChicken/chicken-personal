# 👩‍💻 commit type

### commit 的類型：

- feat： 新功能、新特性
- fix： 修改 bug
- perf： 更改代碼，以提高性能（在不影響代碼內部行為的前提下，對程式性能進行優化）
- refactor： 代碼重構（重構，在不影響代碼內部行為、功能下的代碼修改）
- docs： 文件修改
- style： 代碼格式修改， 注意不是 css 修改（例如分號修改）
- test： 測試用例新增、修改
- build： 影響項目構建或依賴項修改
- revert： 恢復上一次提交
- ci： 持續集成相關文件修改
- chore： 其他修改（不在上述類型中的修改）
- release： 發佈新版本
- workflow： 工作流相關文件修改
