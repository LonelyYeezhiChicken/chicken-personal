# 📊 報表自動化系統

本章節解析 SPC 系統的「自動化報表中心」。在管理成千上萬監控項時，系統核心價值在於「主動派送」與「智能聚焦」。

## 1. 高效預渲染與緩存策略

針對全廠品質日報，系統採用定時觸發任務：
- **凌晨任務**：自動計算前一天的指標並渲染 PNG/PDF 存入緩存。
- **秒開體驗**：用戶載入報表僅需約 $100\text{ms}$。

## 2. 動態報表範本引擎

- **變量填充**：支援嵌入動態標籤（如 `{CUR_CPK}`）。
- **多格式輸出**：導出為 PDF、Excel 或 HTML。

## 3. 焦點報告與異常驅動 (Focus Reporting)

專家級報表僅顯示「異常項」。系統自動篩選：
- $C_{pk} < 1.0$ 的監控項。
- $OOC$ 次數環比增長超過 $50\%$ 的項。
- 已發生異常但長未處理 (ACK) 的項。

## 4. 派送系統與權限隔離

- **異構通報**：透過 Email 或專屬通訊軟體派送。
- **數據權限**：根據組織部門裁切報表內容，確保數據安全。

## 5. 領域專家思維：報表是決策引擎

專家深知，一份好的報表應引導下一步行動。
- **行動鏈結**：嵌入「深連結 (Deep Link)」，讓工程師能直接跳轉回系統進行下鑽分析。
- **反饋循環**：不斷優化焦點報告的過濾閾值，實現精準管理。
