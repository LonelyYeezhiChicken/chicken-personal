# 📊 管制界限與規格界限的博弈

本章節解析 SPC 系統中最常被混淆的兩大界限系統。理解這兩者的差異，是邁向製程穩定性專家的必經之路。

## 1. 管制界限 (Control Limits) —— 製程的聲音

管制界限 ($UCL/CL/LCL$) 是由「歷史數據的統計特徵」所決定的，它不代表任何外在需求。

### 1.1 $3\sigma$ 的統計與經濟含義
- **定義**：設定在平均值 $\mu \pm 3\sigma$。
- **機率含義**：在常態分佈下，有 $99.73\%$ 的數據點應落在界限內。
- **設計決策**：這是統計學上的「經濟平衡點」。
    - **型一錯誤 ($\alpha$)**：虛警（製程正常卻報警）。
    - **型二錯誤 ($\beta$)**：漏警（製程異動卻未報警）。
- **專家見解**：$3\sigma$ 是在「過度干預代價」與「延誤改善損失」之間取得的最佳平衡。

## 2. 規格界限 (Specification Limits) —— 客戶的聲音

規格界限 ($USL/Target/LSL$) 是由「工程設計或客戶需求」所決定的，與製程的統計數據無關。

### 2.1 規格的核心意義
- **職責**：判斷產品「合格」或「廢品」。
- **設計哲學**：代表了客戶能忍受的物理極限。

## 3. 兩者之間的博弈與衝突場景

### 3.1 理想狀態：界限窄於規格
- **現象**：$6\sigma$ 寬度遠小於 $(USL - LSL)$。
- **結論**：製程不僅受控，且具備極高的 $C_{pk}$。

### 3.2 警示狀態：界限寬於規格
- **現象**：管制界限超出了規格界限。
- **結論**：即便製程「受控」，產出的產品仍會包含廢品。
- **專家行動**：這屬於「製程能力不足」。

### 3.3 虛警狀態：點出界但規格內
- **現象**：數據點落在管制界限外，但在規格內。
- **結論**：產品仍是合格的，但製程已經「不穩定」了。

## 4. 領域專家思維：界限不可手動調整

- **錯誤行為**：為了減少報警，手動將管制界限調寬。
- **正確行為**：管制界限必須由統計引擎自動更新。
